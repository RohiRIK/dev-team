# Test Case Design

A test case is a set of conditions or variables under which a tester will determine whether a system under test satisfies requirements or works correctly. The process of developing test cases can also help find problems in the requirements or design of an application.

## Test Case Design Techniques

-   **Equivalence Partitioning:** Divides the input data into partitions of equivalent data from which test cases can be derived.
-   **Boundary Value Analysis:** Focuses on the values at the boundaries of the equivalence partitions.
-   **Decision Table Testing:** A systematic approach where rules are represented as a table, and test cases are derived from the table.
-   **State Transition Testing:** Used to test the behavior of a system that can be in different states.
-   **Use Case Testing:** Helps to identify test cases that cover the entire system, on a transaction by transaction basis from the start to the end.
