# Resume Best Practices for Automation Engineers

This document provides a knowledge base for creating a professional and effective resume, tailored to the user's goal of landing a role as an Automation Engineer (**G1**). It serves as the primary guide for the `cv-generator` agent.

---

## 1. The Goal: Pass the 15-Second Scan

A recruiter or hiring manager will likely spend only 15-30 seconds on the first scan of your resume. Our goal is to make your skills and impact immediately obvious and compelling.

---

## 2. Resume Structure

The `cv-generator` agent should follow this standard, effective structure.

### Section 1: Contact Information
- **Content:** Full Name, Phone Number, Email, LinkedIn URL, GitHub Profile URL.
- **Instruction:** This should be clean and easy to find at the very top of the page.

### Section 2: Summary
- **Content:** A 2-3 sentence summary that highlights your key value proposition. It should be tailored to the specific job you are applying for.
- **Example:** "Cloud Security and AI Automation Engineer with X years of experience in designing and implementing secure, scalable automation solutions. Seeking to leverage expertise in n8n, Python, and Azure to drive efficiency as an Automation Engineer."

### Section 3: Skills
- **Content:** A categorized list of your technical skills.
- **Instruction:** This is one of the most important sections. The agent must pull this information directly from the **Skills Development** section of `TELOS.md` (**SdJ1-SdJ5**).
- **Example Categorization:**
  - **Automation & Orchestration:** n8n, Python, GitHub Actions, Azure Automation
  - **Cloud & Security:** Azure, Entra ID, Intune, Docker, Terraform, MCAS
  - **Languages:** Python, JavaScript, PowerShell, Bash
  - **AI & Machine Learning:** AI Agents, Prompt Engineering, LangChain

### Section 4: Professional Experience
- **Content:** For each role, list the Title, Company, and Dates.
- **Instruction:** Use 3-5 bullet points per role. **Crucially, focus on achievements, not just responsibilities.** Start every bullet point with a strong action verb (see `action_verbs.md`).
- **Quantify Everything:** Instead of "*Responsible for automating tasks*," write "*Automated the user offboarding process, reducing manual work by an estimated 10 hours per month.*" This demonstrates impact.

### Section 5: Projects
- **Content:** A curated list of your most impressive and relevant personal projects.
- **Instruction:** The agent should select 3-4 key projects from the `Projects` section of `TELOS.md`. For an Automation Engineer role, projects like the **n8n-Workflows collection (Prtt2)**, the **secure-chatbot-ms-tenants (Prtd7)**, and the **Home Lab on Docker Swarm (Prtt9)** are excellent choices.
- **For each project:** Include the project name, a 1-2 sentence description, and a link to the GitHub repository.

### Section 6: Education
- **Content:** List any relevant degrees or certifications (like your cybersecurity course, **H20**).

---

## 3. Formatting

- **Length:** Keep it to one page.
- **Format:** Always generate the final output as a PDF.
- **Clarity:** Use clear headings, consistent formatting, and plenty of white space.
