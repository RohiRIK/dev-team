# Tool Creation

Tool files define the tools that an agent can use to perform its tasks. They should be well-documented and easy to understand. The following are some guidelines for creating effective tool files:

-   **Use Markdown:** Tool files should be written in Markdown.
-   **Provide a Clear Description:** Provide a clear and concise description of the tool.
-   **Define the Input and Output:** Define the input and output of the tool.
-   **Provide Examples:** Provide examples of how to use the tool.
-   **Keep it Updated:** Keep the tool files up-to-date with the latest changes to the tool.
