## How to Contribute

### 1. Reporting Bugs

If you find a bug, please open an issue on GitHub. Before opening a new issue, please check if a similar issue already exists.

- **Provide clear steps to reproduce the bug.**
- **Include expected and actual behavior.**
- **Mention your operating system, browser, and any relevant software versions.**

### 2. Suggesting Enhancements

We love new ideas! If you have a suggestion for an enhancement or a new feature:

- **Open an issue** to discuss your idea first. This helps ensure that your contribution aligns with the project's goals.
- **Clearly describe the enhancement** and why you think it would be valuable.

### 3. Making Code Contributions

1.  **Fork the repository** and clone it to your local machine.
2.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/your-feature-name
    # or
    git checkout -b bugfix/your-bug-fix
    ```
3.  **Make your changes.** Ensure your code adheres to the project's coding style and conventions.
4.  **Write tests** for your changes. All new features and bug fixes should have corresponding tests.
5.  **Run tests** to ensure everything is working correctly.
6.  **Write clear, concise commit messages.** (See `git_best_practices.md` for guidance).
7.  **Push your branch** to your forked repository.
8.  **Open a Pull Request (PR)** to the `main` branch of the original repository.

---

## Pull Request Guidelines

- **One feature/bug fix per PR:** Keep your PRs focused.
- **Descriptive PR title and description:** Explain your changes thoroughly.
- **Reference issues:** Link your PR to any relevant issues (e.g., `Closes #123`).
- **Pass all CI checks:** Ensure your code passes all automated tests and linting.
- **Request a review:** Ask for a review from a maintainer.
