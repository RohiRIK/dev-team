# Mandatory Quality Gates for All Agents

🚨 **CRITICAL: These are MANDATORY checkpoints - NOT optional suggestions** 🚨

## Pre-Execution Checklist (MANDATORY)

Before starting ANY task, you MUST verify:

- [ ] **Requirements Clear?** - Do I understand exactly what needs to be done?
- [ ] **Dependencies Identified?** - What do I need from other agents/systems?
- [ ] **MCP Tools Available?** - Are all required tools accessible?
- [ ] **Context Gathered?** - Do I have all necessary background information?
- [ ] **Success Criteria Defined?** - How will I know when I'm done?

**If ANY checkbox is unchecked, STOP and gather missing information first.**

---

## Mid-Work Validation (MANDATORY)

At 50% completion or after each major step, you MUST check:

- [ ] **Matches Requirements?** - Does current work align with original request?
- [ ] **No Scope Creep?** - Am I staying focused on the task?
- [ ] **Quality Standards Met?** - Is the work meeting expected quality?
- [ ] **Blockers Identified?** - Are there any issues preventing progress?
- [ ] **Still on Track?** - Is the approach working or should I pivot?

**If validation fails, document the issue and adjust approach immediately.**

---

## Pre-Completion Review (MANDATORY)

Before marking ANY task as complete, you MUST verify:

- [ ] **Tested?** - Has the work been tested/validated?
- [ ] **Documented?** - Are changes documented appropriately?
- [ ] **Error Handling?** - Are edge cases and errors handled?
- [ ] **Dependencies Updated?** - Have dependent systems/agents been notified?
- [ ] **Summary Created?** - Is task summary written in `_summaries/`?
- [ ] **Code Quality?** - Does code follow best practices and standards?
- [ ] **Security Checked?** - Are there any security vulnerabilities?
- [ ] **Performance Acceptable?** - Does it meet performance requirements?

**Task is NOT complete until ALL checkboxes are checked.**

---

## Failure to Follow = Invalid Work

If quality gates are skipped, the work is considered **incomplete and invalid**.

Every agent must treat these gates as **hard requirements**, not suggestions.
