# OWASP Top 10:2021

The OWASP Top 10 is a standard awareness document for developers and web application security, representing a broad consensus about the most critical security risks to web applications. The latest official version is the OWASP Top 10:2021, released on September 24, 2021.

## The OWASP Top 10:2021 List

1.  **A01:2021-Broken Access Control:** This category moved up from fifth position and is the most serious web application security risk, with data indicating that, on average, 3.81% of applications tested had one or more Common Weakness Enumerations (CWEs) related to it.
2.  **A02:2021-Cryptographic Failures:** Formerly known as Sensitive Data Exposure, this category shifts up to the second position. The renewed focus is on failures related to cryptography that often lead to sensitive data exposure or system compromise.
3.  **A03:2021-Injection:** This category slides down to the third position. It encompasses various forms of injection, including Cross-site Scripting (XSS), which is now part of this category.
4.  **A04:2021-Insecure Design:** A new category for 2021, it focuses on risks related to design flaws, emphasizing the need for threat modeling, secure design patterns, and principles.
5.  **A05:2021-Security Misconfiguration:** This category moves up one spot from the 2017 list and now includes the previous XML External Entities (XXE) category.
6.  **A06:2021-Vulnerable and Outdated Components:** Previously titled "Using Components with Known Vulnerabilities," this category moved up from ninth position in 2017.
7.  **A07:2021-Identification and Authentication Failures:** This category addresses issues related to user identity, authentication, and session management.
8.  **A08:2021-Software and Data Integrity Failures:** This is a new category for 2021, focusing on assumptions related to software updates, critical data, and CI/CD pipelines without verifying integrity. Insecure Deserialization is now part of this larger category.
9.  **A09:2021-Security Logging and Monitoring Failures:** Previously A10:2017-Insufficient Logging & Monitoring, this category moved up from the previous list.
10. **A10:2021-Server-Side Request Forgery (SSRF):** This is a new category for 2021.

## Updates

The OWASP Top 10 is updated periodically, typically every three to four years. The OWASP team is planning to announce the release of the OWASP Top 10:2025 at the OWASP Global AppSec Conference in Washington D.C. in the first week of November 2025.
