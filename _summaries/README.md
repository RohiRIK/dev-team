# Task Summaries

This folder contains summaries of completed tasks, bug fixes, features, and research. Each summary documents what was done, how it was verified, and what comes next.

## Purpose

- 📝 Document completed work
- 🔍 Track project history
- 💡 Capture learnings and insights
- 🚀 Speed up onboarding
- 🔄 Prevent repeating mistakes

## Naming Convention

```
YYYY-MM-DD-brief-task-description.md
```

**Examples:**
- `2025-10-27-fix-react-hooks-errors.md`
- `2025-10-27-implement-oauth2-authentication.md`
- `2025-10-27-playwright-browser-installation-fix.md`

## Required Format

Each summary must include:

1. **Task Summary Title**
2. **Date**
3. **Objective** - What was the goal?
4. **Actions Taken** - What did you do?
5. **Verification** - How did you test it?
6. **Current Status** - What's the final state?
7. **Next Steps** - What should happen next?
8. **Files Modified/Created** - Which files changed?
9. **Key Learnings** - What did you learn?

See `.gemini/agents/_shared/knowledge/task-summary-documentation.md` for complete template and guidelines.

## When to Create Summaries

✅ After completing:
- Feature implementations
- Bug fixes
- Research or investigations
- Refactoring tasks
- Multi-step work
- Any significant task
